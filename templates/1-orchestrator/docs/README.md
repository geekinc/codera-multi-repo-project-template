# docs/

Markdown documentation for this project lives here. Each `.md` file in this
directory surfaces in the Codera orchestrator card's **Docs** view, so add
documents freely — `ARCHITECTURE.md` is the canonical architecture reference,
but anything you want the team (and sub-agents) to read belongs here too.

## Conventions

- **Markdown only** for now — `.md` files are the only format the Docs view reads.
- Keep `ARCHITECTURE.md` as the source-of-truth architecture document.
- Use clear, descriptive filenames (e.g. `API-CONTRACTS.md`, `RUNBOOK.md`); the
  filename is what the Docs view shows in its list.
